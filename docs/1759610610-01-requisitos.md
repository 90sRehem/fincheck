# Requisitos

## 🎯 Requisitos Funcionais

| Id   | Requisito                       | Descrição                                                                                          | Observações                                                                                                                     |
| ---- | ------------------------------- | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| RF01 | Contas e Carteiras              | O usuário deve poder criar múltiplas carteiras (ex: Conta Bancária, Dinheiro, Cartão de Crédito).  | Cada carteira tem um saldo inicial, tipo (corrente, poupança, crédito) e moeda.                                                 |
| RF02 | Transações                      | O usuário deve registrar transações de entrada (receita) e saída (despesa).                        | Transações devem conter: valor, categoria, descrição, data e meio de pagamento. Deve ser possível editar ou excluir transações. |
| RF03 | Categorias                      | O usuário pode criar categorias personalizadas (ex: Alimentação, Transporte).                      | {{observações}}                                                                                                                 |
| RF03 | Metas e Objetivos               | O usuário pode criar metas de economia e orçamentos mensais por categoria.                         | O sistema deve alertar quando o gasto ultrapassar o orçamento definido.                                                         |
| RF04 | Relatórios e Dashboards         | Exibir gráficos de evolução de saldo, despesas por categoria, receitas mensais, etc.               |                                                                                                                                 |
| RF05 | Recorrências                    | Criar transações recorrentes (ex: salário, aluguel).                                               |                                                                                                                                 |
| RF06 | Sincronização Bancária (futuro) | Permitir integração com instituições financeiras (via importação de extrato, open finance ou CSV). |                                                                                                                                 |
| RF07 | Autenticação                    | O usuário deve ter autenticação segura (email/senha ou OAuth).                                     |                                                                                                                                 |
| RF08 | Notificações                    | Alertas de metas atingidas, orçamentos estourados, e lembretes de contas recorrentes.              |                                                                                                                                 |

## ⚡ Requisitos Não Funcionais

| ID    | Categoria           | Descrição                                                                          |
| ----- | ------------------- | ---------------------------------------------------------------------------------- |
| RNF01 | Segurança           | Armazenar dados de forma segura e criptografada (principalmente senhas e tokens).  |
| RNF02 | Privacidade         | Dados do usuário são privados e não compartilhados.                                |
| RNF03 | Disponibilidade     | O sistema deve funcionar mesmo offline (sincronizando depois).                     |
| RNF04 | Desempenho          | Consultas de transações e relatórios devem ser rápidas mesmo com muitos registros. |
|       | Escalabilidade      | Permitir múltiplos usuários sem degradação de performance.                         |
|       | Auditabilidade      | Registrar logs de alterações e exclusões de transações.                            |
|       | Extensibilidade     | Domínio preparado para novas features (ex: investimentos, dívidas, cashback).      |
|       | Internacionalização | Internacionalização                                                                |
